Development

Initially

The plan was to first build a TiltBrush like tool and then develop the game on top of that. However, by the time I had figured out how to develop the TiltBrush with proper physics, time had become my enemy and I realized I could not defeat it--at least not this time. Therefore I will outline to you all now what it is I have done so far in the project.

Drawing a Line

The very first thing I had to figure out was how to draw a line in 3D. This seemed quite challenging at first, but I found plenty of online resources to show me that Unity comes with a component named LineRenderer which I could use to draw on any plane and axis.

Learning to draw a realistic Line

Although I had learned to draw a line, I managed to catch that it was not a true 3D line. That is, it was not visible from all angles and it did not look very realistic. I searched for LineRenderer alternatives and found a blog that contained some vector programming source code. I then spent some time looking for ways I could change the source code in a way that tailored my needs. I eventually found a helpful Youtube video. This was definitely the most difficult part of the project (I think even more difficult than what I had in mind for developing an accuracy measure for the drawings).

Textures

The next part was to get some textures and be able to draw using them.

ColorPicker

The color picker I found in the asset store was quite bland but I didn’t have time to worry about that. Instead I focused on changing it up so that I could then apply the controls to the VR controllers and also so that I could then used the controllers to apply the color change to my lines properly.

That’s where I ended. My plan is to eventually finish the entire project (although by no means will I have this done for my school deadline). But it is something I would like to finish regardless.
